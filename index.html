MOTION PRINCIPLES

Purposeful — Every motion means something
No decorative animation. Every transition communicates state change. A card sliding in means data has loaded. A colour shifting from amber to green means a fault has been resolved. Motion is a language, not decoration.
Responsive — Interaction acknowledged within 16ms
Every tap, click, or swipe receives immediate visual feedback. Micro-animations (scale, glow, border colour) confirm selection before the data action completes. The system feels instantaneous even when it is not.
Elegant — Cubic-bezier easing, not linear
All transitions use ease-out or custom cubic-bezier curves. Nothing starts or stops abruptly. Scroll behaviour is smooth. Page transitions use fade+translate — 300ms maximum. The interface feels like a living thing, not a static document.
Restrained — Not every component animates
Data rows, table cells, and status labels update without animation — only the values change. Animation is reserved for navigation, modals, alerts, and critical state changes. This keeps the interface feeling professional rather than theatrical.

Staff Interface

AMARA OPS

AMARA Ops is the operational nerve centre for the 25,000+ staff who support the grid, the customers, and the company. It is desktop-only — designed for focused, professional use across wide screens. NEXUS AI is the primary operator. Human staff are the exception handler. The interface is structured around that reality: AI surfaces everything, humans decide on the edge cases.

TICKET SYSTEM — AI TRIAGED

Every customer complaint, fault alert, and grid anomaly generates a ticket. NEXUS analyses each ticket on arrival — classifying severity, suggesting root cause, and recommending resolution steps. Staff see tickets pre-sorted, pre-diagnosed, and pre-recommended. Human effort is applied only where AI certainty falls below 90%.

Security & Authentication

AUTH DESIGN

Authentication is the first and most critical fraud prevention layer. Each platform uses a different authentication model matched to its user type, risk profile, and device context. The design specifically addresses Nigeria's known fraud vectors: SIM-swap attacks, social engineering, and identity theft.

CONNECT — CONSUMER

📱
Phone Number
Nigerian number. 11-digit NCC format. Used as primary identity — not username.
🔢
OTP Verification
6-digit OTP via SMS. 120-second expiry. Max 3 attempts. Anti-SIM-swap: OTP delivery blocked if SIM changed within 48 hours.
🪪
NIN Linking
National Identity Number linked at account creation. Not used for daily login but required for recovery and family account management.
👆
Biometric (subsequent logins)
Face ID / fingerprint on supported devices. PIN fallback. Daily reauth with biometric — no persistent sessions that can be hijacked.

OPS — STAFF

🆔
Staff ID + Password
Unique AMARA-issued Staff ID (AO-[REGION]-[ROLE]-[SERIAL]). Password min 14 chars. Enforced rotation every 90 days.
🔑
Hardware 2FA Token
TOTP via Yubikey or Google Authenticator. Every login requires hardware token. No SMS-based 2FA for staff — SIM-swap risk is too high.
🖥️
Device Registration
Staff can only log in from pre-registered AMARA-issued devices. New device login requires IT department approval and re-verification.
📋
Role-Based Access Control
Every action is permission-gated. Customer service agents cannot access billing override. Only compliance officers see full payment history. NEXUS logs all access events.

COMMAND — MANAGEMENT

👁️
Biometric + Password
Face recognition + iris scan at Command Centre terminals. Password as secondary. No remote login permitted for top-tier management accounts.
📍
Location Verification
Command access only from geofenced physical Command Centre locations. Any off-location login attempt triggers immediate security alert and session block.
🤝
Dual Authorisation
Any action affecting national grid configuration requires approval from two management-level accounts simultaneously. Single-person override is architecturally impossible.
🔐
Full Session Recording
Every Command session is fully recorded, timestamped, and encrypted to immutable storage. No action taken at Command level is unaccountable.

Data Architecture

DATABASE ARCHITECTURE

50 million customer records. 500 million annual transactions. 50,000 Smart Terminals reporting every 15 minutes. The AMARA data architecture is designed to scale without degradation — using a combination of purpose-built databases for each data type, sharded for geographic distribution, and replicated across a minimum of three data centres.

CUSTOMER ID FORMAT

Every customer account is assigned a unique AMARA Customer ID at registration. The format encodes geographic zone and account type for immediate context without database lookup.
